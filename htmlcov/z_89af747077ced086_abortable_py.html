1"""Abortable Tasks. 

2 

3Abortable tasks overview 

4========================= 

5 

6For long-running :class:`Task`'s, it can be desirable to support 

7aborting during execution. Of course, these tasks should be built to 

8support abortion specifically. 

9 

10The :class:`AbortableTask` serves as a base class for all :class:`Task` 

11objects that should support abortion by producers. 

12 

13* Producers may invoke the :meth:`abort` method on 

14 :class:`AbortableAsyncResult` instances, to request abortion. 

15 

16* Consumers (workers) should periodically check (and honor!) the 

17 :meth:`is_aborted` method at controlled points in their task's 

18 :meth:`run` method. The more often, the better. 

19 

20The necessary intermediate communication is dealt with by the 

21:class:`AbortableTask` implementation. 

22 

23Usage example 

24------------- 

25 

26In the consumer: 

27 

28.. code-block:: python 

29 

30 from celery.contrib.abortable import AbortableTask 

31 from celery.utils.log import get_task_logger 

32 

33 from proj.celery import app 

34 

35 logger = get_logger(__name__) 

36 

37 @app.task(bind=True, base=AbortableTask) 

38 def long_running_task(self): 

39 results = [] 

40 for i in range(100): 

41 # check after every 5 iterations... 

42 # (or alternatively, check when some timer is due) 

43 if not i % 5: 

44 if self.is_aborted(): 

45 # respect aborted state, and terminate gracefully. 

46 logger.warning('Task aborted') 

47 return 

48 value = do_something_expensive(i) 

49 results.append(y) 

50 logger.info('Task complete') 

51 return results 

52 

53In the producer: 

54 

55.. code-block:: python 

56 

57 import time 

58 

59 from proj.tasks import MyLongRunningTask 

60 

61 def myview(request): 

62 # result is of type AbortableAsyncResult 

63 result = long_running_task.delay() 

64 

65 # abort the task after 10 seconds 

66 time.sleep(10) 

67 result.abort() 

68 

69After the `result.abort()` call, the task execution isn't 

70aborted immediately. In fact, it's not guaranteed to abort at all. 

71Keep checking `result.state` status, or call `result.get(timeout=)` to 

72have it block until the task is finished. 

73 

74.. note:: 

75 

76 In order to abort tasks, there needs to be communication between the 

77 producer and the consumer. This is currently implemented through the 

78 database backend. Therefore, this class will only work with the 

79 database backends. 

80""" 

81from celery import Task 

82from celery.result import AsyncResult 

83 

84__all__ = ('AbortableAsyncResult', 'AbortableTask') 

85 

86 

87""" 

88Task States 

89----------- 

90 

91.. state:: ABORTED 

92 

93ABORTED 

94~~~~~~~ 

95 

96Task is aborted (typically by the producer) and should be 

97aborted as soon as possible. 

98 

99""" 

100ABORTED = 'ABORTED' 

101 

102 

103class AbortableAsyncResult(AsyncResult): 

104 """Represents an abortable result. 

105 

106 Specifically, this gives the `AsyncResult` a :meth:`abort()` method, 

107 that sets the state of the underlying Task to `'ABORTED'`. 

108 """ 

109 

110 def is_aborted(self): 

111 """Return :const:`True` if the task is (being) aborted.""" 

112 return self.state == ABORTED 

113 

114 def abort(self): 

115 """Set the state of the task to :const:`ABORTED`. 

116 

117 Abortable tasks monitor their state at regular intervals and 

118 terminate execution if so. 

119 

120 Warning: 

121 Be aware that invoking this method does not guarantee when the 

122 task will be aborted (or even if the task will be aborted at all). 

123 """ 

124 # TODO: store_result requires all four arguments to be set, 

125 # but only state should be updated here 

126 return self.backend.store_result(self.id, result=None, 

127 state=ABORTED, traceback=None) 

128 

129 

130class AbortableTask(Task): 

131 """Task that can be aborted. 

132 

133 This serves as a base class for all :class:`Task`'s 

134 that support aborting during execution. 

135 

136 All subclasses of :class:`AbortableTask` must call the 

137 :meth:`is_aborted` method periodically and act accordingly when 

138 the call evaluates to :const:`True`. 

139 """ 

140 

141 abstract = True 

142 

143 def AsyncResult(self, task_id): 

144 """Return the accompanying AbortableAsyncResult instance.""" 

145 return AbortableAsyncResult(task_id, backend=self.backend) 

146 

147 def is_aborted(self, **kwargs): 

148 """Return true if task is aborted. 

149 

150 Checks against the backend whether this 

151 :class:`AbortableAsyncResult` is :const:`ABORTED`. 

152 

153 Always return :const:`False` in case the `task_id` parameter 

154 refers to a regular (non-abortable) :class:`Task`. 

155 

156 Be aware that invoking this method will cause a hit in the 

157 backend (for example a database query), so find a good balance 

158 between calling it regularly (for responsiveness), but not too 

159 often (for performance). 

160 """ 

161 task_id = kwargs.get('task_id', self.request.id) 

162 result = self.AsyncResult(task_id) 

163 if not isinstance(result, AbortableAsyncResult): 

164 return False 

165 return result.is_aborted()
