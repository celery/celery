1"""Worker Remote Control Client. 

2 

3Client for worker remote control commands. 

4Server implementation is in :mod:`celery.worker.control`. 

5There are two types of remote control commands: 

6 

7* Inspect commands: Does not have side effects, will usually just return some value 

8 found in the worker, like the list of currently registered tasks, the list of active tasks, etc. 

9 Commands are accessible via :class:`Inspect` class. 

10 

11* Control commands: Performs side effects, like adding a new queue to consume from. 

12 Commands are accessible via :class:`Control` class. 

13"""

38def flatten_reply(reply): 

39 """Flatten node replies. 

40 

41 Convert from a list of replies in this format:: 

42 

43 [{'a@example.com': reply}, 

44 {'b@example.com': reply}] 

45 

46 into this format:: 

47 

48 {'a@example.com': reply, 

49 'b@example.com': reply} 

50 """ 

51 nodes, dupes = {}, set() 

52 for item in reply: 

53 [dupes.add(name) for name in item if name in nodes] 

54 nodes.update(item) 

55 if dupes: 

56 warnings.warn(DuplicateNodenameWarning( 

57 W_DUPNODE.format( 

58 pluralize(len(dupes), 'name'), ', '.join(sorted(dupes)), 

59 ), 

60 )) 

61 return nodes
